# 会話からスラッシュコマンドを抽出

現在のセッションの会話履歴を分析し、繰り返し行われた操作やワークフローをスラッシュコマンドとして抽出してください。

## 分析対象

1. **繰り返し実行されたタスク**
   - 同じ種類の操作を複数回依頼された場合
   - 「また〜して」「前と同じように」などの指示

2. **複数ステップのワークフロー**
   - 毎回同じ順序で実行される一連の操作
   - 定型的な手順（ビルド→テスト→コミットなど）

3. **カスタマイズされた操作**
   - 特定のオプションやフォーマットを常に指定する操作
   - プロジェクト固有の作業手順

## 実行手順

1. **会話履歴の分析**
   - 現在のセッション内の会話を振り返る
   - コマンド化できそうなパターンを特定

2. **コマンド候補の提示**
   ```md
   ## コマンド候補

   ### /[コマンド名]
   - **説明**: 〜
   - **抽出元**: 会話の〇〇で実行した操作
   - **含まれるステップ**:
     1. 〜
     2. 〜
   ```

3. **ユーザー確認**
   - コマンド名、説明、手順をユーザーに確認
   - 調整や追加の指示を受け付ける

4. **コマンドファイルの作成**
   - `.codex/prompts/[コマンド名].md` として保存
   - 既存のコマンドと命名が重複しないよう確認

## コマンドファイルのテンプレート

```md
---
description: <English description>
---

# [コマンドの日本語タイトル]

[コマンドの目的・概要を1-2文で説明]

## 実行手順

1. [ステップ1]
2. [ステップ2]
...

## オプション（あれば）

- 引数の説明と使用例

## 注意事項（あれば）

- 実行前の確認事項
- 制約条件
```

## 注意事項

- 一度きりの特殊な操作はコマンド化しない
- 汎用的すぎる操作（「ファイルを読む」など）は既存ツールで十分
- コマンド名は短く、動詞から始める（例: `build-and-test`, `deploy-staging`）
